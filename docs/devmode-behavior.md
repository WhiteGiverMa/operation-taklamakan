# DevMode 行为说明

## 文档目的

这份文档描述**开发者模式（DevMode）的功能、激活方式和使用场景**。

DevMode 是运行时调试工具，用于加速开发验证、快速测试游戏状态和调试可视化。

---

## 一、功能概述

DevMode 是一个运行时命令执行系统，允许开发者在游戏运行时通过命令行或 UI 界面直接操作游戏状态。

主要用途：

- 快速调整资源（金币、血量）
- 切换调试可视化（碰撞框、路径、攻击范围）
- 生成或清除敌人
- 跳过波次或章节推进
- 通过 MCP 接口进行自动化验证

**警告**：DevMode 仅用于开发调试，正式发布时应禁用或移除。

---

## 二、激活方式

### 1. 设置中开启

1. 在游戏中打开**设置菜单**
2. 找到**开发者模式**选项
3. 将其切换为**开启**状态

DevMode 的启用状态由 `SettingsManager.dev_mode_enabled` 管理，设置会持久化保存。

### 2. 唤出开发者面板

启用 DevMode 后，按下**反引号键（`）** 即可唤出开发者面板。

再次按下反引号键可关闭面板。

> 如果键盘布局没有反引号键，可在 `input_map` 中修改绑定。

---

## 三、面板标签页功能

开发者面板包含四个标签页，分别对应不同功能类别。

### 1. 资源标签（Resources）

用于快速调整游戏资源状态。

| 功能 | 说明 |
|------|------|
| +100 / +1000 / +10000 按钮 | 快速增加对应数量金币 |
| 自定义输入 + "添加" 按钮 | 输入任意数量金币后点击添加 |
| 满血恢复 | 将陆行舰恢复至满血 |
| 跳过当前波 | 跳过当前波间期或开始下一波 |
| 跳过当前章 | 直接推进到下一章 |

### 2. 生成标签（Spawn）

用于控制敌人生成。

| 功能 | 说明 |
|------|------|
| 敌人类型选择 | OptionButton 选择 tank / dog / boss |
| 数量 | SpinBox 指定生成数量（1-50） |
| 生成敌人 | 在生成点附近生成选定类型和数量的敌人 |
| 清空所有敌人 | 清除场景中所有敌人 |

### 3. 调试标签（Debug）

用于切换调试可视化。

| 开关 | 说明 |
|------|------|
| 碰撞框显示 | 显示所有碰撞体的边界框 |
| 路径显示 | 显示敌人的移动路径 |
| 攻击范围显示 | 显示炮塔和敌人的攻击范围 |

### 4. 控制台标签（Console）

用于直接输入命令。

| 功能 | 说明 |
|------|------|
| 命令输入框 | 输入命令并回车执行 |
| 历史记录 | 显示已执行的命令和返回结果 |
| 上下导航 | 使用上下箭头浏览历史命令 |

---

## 四、命令行使用方法

在控制台标签中可直接输入命令。命令格式：

```
<命令名> [参数1] [参数2] ...
```

示例：

```
add_gold 1000
spawn_enemy tank 5
toggle_collision
```

输入 `help` 可查看所有可用命令及其用法。

详细命令列表见 `docs/devmode-commands.md`。

## 五、MCP 集成说明

DevMode 可通过 Godot MCP 接口进行远程调用，用于自动化验证。

### 调用方式

通过 MCP 的 `eval` 操作调用 `DevMode.execute()`：

```gdscript
DevMode.execute("add_gold 500")
```

### 典型场景

1. **自动化测试准备**
   - 设置初始金币
   - 恢复船体血量
   - 清除残留敌人

2. **状态快速验证**
   - 跳过波次进入特定阶段
	- 推进到目标章节
   - 生成特定敌人类型

3. **调试可视化控制**
   - 开启碰撞框检查物理边界
   - 显示攻击范围验证瞄准逻辑

### MCP 示例调用

```json
{
  "operation": "eval",
  "code": "return DevMode.execute('add_gold 1000')"
}
```

## 六、注意事项

1. **DevMode 状态检查**：未启用时执行命令会返回提示信息，不会产生副作用。

2. **命令参数验证**：参数错误时会返回用法提示，不会导致崩溃。

3. **场景依赖**：部分命令依赖特定场景状态（如 `heal` 需要场景中存在陆行舰），在错误的场景下会返回错误信息。

4. **波次状态敏感**：`skip_wave` 和 `spawn_wave` 对 WaveManager 当前状态有要求，不同状态下的行为略有不同。

5. **敌人类型限制**：`spawn_enemy` 仅支持 `tank`、`dog`、`boss` 三种类型，传入其他类型会返回错误。
