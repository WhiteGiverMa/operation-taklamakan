# DevMode 命令参考

## 文档目的

这份文档列出 DevMode 的所有可用命令、参数说明和使用示例。

---

## 命令列表

### 调试开关命令

| 命令 | 参数 | 描述 | 示例 |
|------|------|------|------|
| `toggle_collision` | 无 | 切换碰撞箱显示状态 | `toggle_collision` |
| `toggle_paths` | 无 | 切换敌人移动路径显示状态 | `toggle_paths` |
| `toggle_ranges` | 无 | 切换攻击范围显示状态 | `toggle_ranges` |

### 资源命令

| 命令 | 参数 | 描述 | 示例 |
|------|------|------|------|
| `add_gold` | `<amount>` | 增加指定数量的金币，amount 必须为正整数 | `add_gold 1000` |
| `set_gold` | `<amount>` | 将金币设置为指定数值，amount 可为 0 或正整数 | `set_gold 5000` |
| `heal` | 无 | 将陆行舰恢复至满血 | `heal` |
| `skip_wave` | 无 | 跳过当前波间期并开始下一波；若战斗未开始则自动启动 | `skip_wave` |
| `skip_chapter` | 无 | 跳过当前章节，推进到下一章 | `skip_chapter` |

### 生成命令

| 命令 | 参数 | 描述 | 示例 |
|------|------|------|------|
| `spawn_enemy` | `<type> [count]` | 在生成点附近生成敌人；type 可选值：tank, dog, boss；count 默认为 1 | `spawn_enemy tank 3` |
| `clear_enemies` | 无 | 清除场景中所有敌人 | `clear_enemies` |
| `spawn_wave` | `[wave_num]` | 开始下一波战斗；若战斗未开始则自动启动会话；wave_num 参数暂不支持直接跳转 | `spawn_wave` |

### 帮助命令

| 命令 | 参数 | 描述 | 示例 |
|------|------|------|------|
| `help` | 无 | 显示所有可用命令及其用法 | `help` |

## 命令详细说明

### toggle_collision

切换碰撞箱显示状态。

- **返回值**：当前状态（"碰撞箱显示已 开启/关闭"）
- **触发事件**：通过 `EventBus.dev_event` 发送 `toggle_collision` 信号
- **适用场景**：调试物理碰撞、检查碰撞体边界

### toggle_paths

切换敌人移动路径显示状态。

- **返回值**：当前状态（"路径显示已 开启/关闭"）
- **触发事件**：通过 `EventBus.dev_event` 发送 `toggle_paths` 信号
- **适用场景**：调试敌人寻路、检查移动轨迹

### toggle_ranges

切换攻击范围显示状态。

- **返回值**：当前状态（"攻击范围显示已 开启/关闭"）
- **触发事件**：通过 `EventBus.dev_event` 发送 `toggle_ranges` 信号
- **适用场景**：调试炮塔瞄准、检查攻击覆盖范围

### add_gold

增加指定数量的金币。

- **参数**：
  - `amount`（必填）：增加的金币数量，必须为正整数
- **返回值**：增加数量和当前金币总数
- **错误情况**：
  - 参数缺失：返回用法提示
  - 参数非正整数：返回错误提示
- **示例**：`add_gold 1000` → "金币增加 1000，当前金币: 1500。"

### set_gold

将金币设置为指定数值。

- **参数**：
  - `amount`（必填）：目标金币数量，可为 0 或正整数；负数自动归零
- **返回值**：设置后的金币数量
- **示例**：`set_gold 5000` → "金币已设置为 5000。"

### heal

将陆行舰恢复至满血。

- **返回值**：恢复的血量值和当前状态
- **错误情况**：
  - 场景中不存在陆行舰（ship 组为空）
  - 陆行舰没有 HealthComponent 节点
  - HealthComponent 类型不匹配
- **示例**：`heal` → "陆行舰已恢复 45 点生命值，当前血量: 100/100。"
- **满血情况**：返回 "陆行舰已经是满血（100/100），无需恢复。"

### skip_wave

跳过当前波间期或开始下一波。

- **行为**：
  - **BETWEEN_WAVES 状态**：跳过波间期，开始下一波
  - **ACTIVE_WAVE 状态**：无法跳过，返回当前战斗状态提示
  - **INACTIVE 状态**：自动启动战斗会话并进入第一波
- **返回值**：操作结果和当前状态
- **示例**：`skip_wave` → "波间期已跳过，当前 WaveManager 状态: BETWEEN_WAVES -> 已开始下一波。"

### skip_chapter

跳过当前章节，推进到下一章。

- **返回值**：推进前后的章节
- **示例**：`skip_chapter` → "章节已从 1 推进到 2。"

### spawn_enemy

在生成点附近生成指定类型和数量的敌人。

- **参数**：
  - `type`（必填）：敌人类型，可选值：`tank`、`dog`、`boss`
  - `count`（可选）：生成数量，默认为 1；非正数自动归 1
- **返回值**：成功生成数量和请求数量
- **错误情况**：
  - 参数缺失：返回用法提示
  - 类型错误：返回可用类型列表
  - 场景资源未配置：返回资源配置错误
  - 未找到生成点：返回场景结构错误
- **示例**：`spawn_enemy tank 3` → "成功生成 3 个 tank，目标请求 3 个。"

### clear_enemies

清除场景中所有敌人。

- **返回值**：清除的敌人数量
- **示例**：`clear_enemies` → "已清除 12 个敌人。"

### spawn_wave

开始下一波战斗。

- **参数**：
  - `wave_num`（可选）：指定波次号（当前实现暂不支持直接跳转）
- **行为**：
  - **INACTIVE 状态**：启动战斗会话并进入第一波
  - **BETWEEN_WAVES 状态**：开始下一波
  - **ACTIVE_WAVE 状态**：无法重复开始
- **返回值**：操作结果和当前波次
- **示例**：`spawn_wave` → "战斗会话已启动，当前波次: 1。"

### help

显示所有可用命令及其用法。

- **返回值**：格式化的命令列表
- **示例**：`help` → 显示包含 12 个命令的列表

---

## 返回值格式说明

所有命令执行后都会返回字符串结果：

- **成功**：返回操作结果描述
- **参数错误**：返回用法提示
- **状态错误**：返回当前状态和错误原因
- **场景错误**：返回缺失的节点或组件信息

---

## 使用建议

1. **调试前准备**：先用 `help` 确认命令列表
2. **资源调整**：用 `add_gold` 和 `heal` 快速进入测试状态
3. **战斗测试**：用 `spawn_enemy` 测试特定敌人行为
4. **可视化调试**：用 `toggle_*` 命令检查物理和逻辑边界
5. **流程跳转**：用 `skip_wave` 和 `skip_chapter` 快速到达目标阶段
